# Sustentacion del parcial: Abstract

El objetivo principal de este parcial consistió en el análisis y modelado de problemas de la vida cotidiana mediante el uso de la Programación Orientada a Objetos (POO). A lo largo del desarrollo de los 20 ejercicios propuestos, se identificaron y aplicaron tres tipos fundamentales de relaciones entre clases: asociación ( Hay conexión, pero las clases son totalmente independientes ), agregación ( Una clase contiene a otra, pero la clase contenida puede existir sola ) y composición ( Una clase depende de otra para existir). El trabajo permite conectar conceptualmente cada ejercicio según la dependencia entre sus entidades; se inicia con asociaciones para vínculos independientes, se avanza hacia la agregación para representar colecciones de objetos que pueden existir por separado, y se finaliza con la composición para estructuras donde las partes dependen totalmente del objeto principal. Este proceso de diseño permitió observar cómo la lógica definida en los diagramas UML se traduce a lenguajes como Java y Python, notando que mientras Java requiere una definición explícita de tipos y estructuras de datos para gestionar las relaciones, Python ofrece una implementación más dinámica y flexible basada en sus listas internas. En conclusión, el desarrollo de este parcial resalta la importancia de realizar un diseño de software correcto antes de la implementación, ya que esto garantiza que el sistema final sea organizado, fácil de entender y mantenible a largo plazo. Identificar adecuadamente estas relaciones es clave para construir aplicaciones de alta calidad que respondan de forma precisa a los requerimientos planteados.
